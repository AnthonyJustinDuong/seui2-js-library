The Problem

We do not want web elements to overlap each other during animations.

Most Basic Example

Using the Seui2 library, you can avoid the element like so.

 Seui('#example1 svg', 'ex1-animate', [
    {
      avoid: '#ex1-avoid',
      start: {x: 873, y: 100},
      end: {x: 33, y: 100},
    },
  ]);

Go here to see an explanation of the code above and get started.

Multiple Animations

You have multiple animations chained together by giving more specification objects. You can avoid a different element for each animation. The animations can start one after each other using '.end+1s' at the front of the begin-value list. Notice that if no start value is specified it uses the previous end position.

 Seui('#example2 svg', 'ex2-animate', [
    { start: {x: 50, y: 150}, end: {x: 325, y: 200} },
    { avoid: '#ex2-avoid2', end: {x: 575, y: 125} },
    { end: {x: 50, y: 150} }
  ],
  {
    avoid: '#ex2-avoid1',
    begin: '.end+1s;1s'
  }
 );

More Options

You can specify the direction around the object, a padding, and a speed of animation. There are also more options that are the same to the SMIL animation attributes. Full documentation can be found here.

 Seui('#example3 svg', 'ex3-animate', [
    {
      start: {x: 50, y: 150}, end: {x: 550, y: 150},
      calcMode: 'spline',
      keySplines: '.75 0 .25 1', keyTimes: '0; 1',
      begin: 'mouseover', dur: '3s',
      restart: 'never'
    },
    {
      end: {x: 50, y: 150},
      bend: 50,
      direction: 'counter-clockwise',
    },
    {
      start: {x: 50, y: 150}, end: {x:49, y: 20},
      bend: 50,
      direction: 'counter clockwise',
    },
  ],
  {
    avoid: '#ex3-avoid',
    begin: '.end+1s;',
    padding: 20,
    speed: 500
  }
 );

Interactive Example

In the example below, you can set the start and end position for the element to animate (blue). Where the coordinates specify the top-left corner of the element. This is the same for the position of the element to avoid (red). You can also change the width and height of both. The library will take this into account and generate the new path. 

  const animator = Seui(...)
  // simply call this function on property changes to re-render the animation
  animator.renderAnimation();
