Seui2 API

Getting Started

Step 1

Add the library to your web page via a script tag.


  <head>
    <title>Seui2 Getting Started</title>
    <script type="text/javascript" src="seui2.js"></script>
  </head>

Step 2

Set up an SVG element. Give the SVG element an id. In the example below, the id is svgId.


  <body>
    <svg id="svgId" width="500" height="500">
    </svg>
  </body>

Step 3

Give the element you wish to animate an id. In the example below, it is toAnimateId.


  <body>
    <svg id="svgId" width="500" height="500">
      <rect id="toAnimateId" x="50" y="100"
        width="50" height="50">
      </rect>
    </svg>
  </body>

Step 4

Give the element you wish to avoid an id. In the example below, it is toAvoidId.


  <body>
    <svg id="svgId" width="500" height="500">
      <rect id="toAnimateId" x="50" y="100"
        width="50" height="50">
      </rect>
      <rect id="toAvoidId" x="150" y="100"
        width="150" height="100" style="fill:red">
      </rect>
    </svg>
  </body>

Step 5

Instantiate a Seui object. The first argument of the constructor is a specifier for the SVG element. The second argument is the id of the element to animate. The third argument is a list of animation specifications. In the example below, the animation specification contains the specifier to the element to avoid, the start point, the end point, and other specifications. Please look here to see what options can be specified.


  <body>
    <svg id="svgId" width="500" height="500">
      <rect id="toAnimateId" x="50" y="100"
        width="50" height="50">
      </rect>
      <rect id="toAvoidId" x="150" y="100"
        width="150" height="100" style="fill:red">
      </rect>
    </svg>
    <script>
      const animator = Seui("#svgId", "toAnimateId", [
        {
          avoid: "#toAvoidId",
          start: {x: 50, y: 100},
          end: {x: 350, y: 150},
          direction: "counter-clockwise",
          begin: "click"
        }
      ])
    </script>
  </body>

Documentation

The Constructor

Seui(svgSpecifier, animateId, animationSpecifications, defaultSpecifications)
  • svgSpecifier
    is a specifier for the SVG element to animate within. The first element to match the specifier will be selected.
  • animateId
    is a string id for the element to be animated.
  • animationSpecifications
    is a list of specification objects. Each object contains animation options, a detailed list can be found below. Each specification object corresponds to one generated animation. 
  • defaultSpecifications
    is a specification object. If a value for an option is not specified in the animationSpecifications for a specific animation, the default value will be taken from here.

Specification Options

An example specification object:

{
  avoid: "#toAvoidId",
  start: {x: 50, y: 100},
  end: {x: 350, y: 150},
  direction: "clockwise",
  begin: "1s"
}

A table of all options:

OptionTypeDescriptionDefault* 
avoid
 string A specifier for the element to avoid. Will not avoid any element and will take a straight line path. 
start
 object
{
'x': number,
'y': number
}		 The x and y coordinates for the starting position. **The previous end value. If not, its current position. 
end
 object
{
'x': number,
'y': number
}		 The x and y coordinates for the ending position. **The next start value. If not, its current position. 
direction
 string The direction to travel around the element: "clockwise" or "counter-clockwise". 'clockwise' 
padding,
paddingTop,
paddingLeft,
paddingBottom,
paddingRight
 number The padding around the element, in pixels. 0 
bend
 number The height of the control point for quadratic curves around the element to avoid. Will bend paths parallel to the boundary. 0 
speed
 number The speed of travel of the animated element in pixels per second. Such that the duration will be '1s'. 
dur
 string The duration of the animation. '1s' 
begin
 string The begining event to begin the animation. 'click' 
endAnimation
 string Constrains the active duration. none 
min, max,
restart,
repeatCount,
repeatDur,
calcMode, values,
keySplines,
keyTimes,
from, to, by
 string Other animation options taken from here.

Notes:

*If not specified in any animation nor default specification.

**"start" and "end" options look for the previous end or the next start, respectively, before taking the one specified in "defaultSpecifications".
